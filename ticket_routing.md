## 一、工单分类体系

### 1.1 一级分类（按问题类型）

| 分类ID | 分类名称 | 定义 | 优先级基准 |
|--------|---------|------|-----------|
| T1 | **功能异常** | 软件无法正常使用、功能报错/崩溃 | P0-P1 |
| T2 | **生成质量问题** | AI生成的PPT内容质量不达标 | P1-P2 |
| T3 | **模板相关** | 模板缺失、模板质量问题、模板使用咨询 | P2-P3 |
| T4 | **性能问题** | 卡顿、加载慢、导出慢、内存占用高 | P1-P2 |
| T5 | **账号与订阅** | 登录问题、付费问题、套餐咨询、退款 | P0-P2 |
| T6 | **使用咨询** | 功能使用指导、操作教程、功能建议 | P3 |
| T7 | **数据安全** | 数据泄露担忧、隐私问题、导出格式 | P1-P2 |
| T8 | **集成与兼容** | 与其他工具集成、文件格式兼容 | P2-P3 |

### 1.2 二级分类（按具体场景）

#### T1 功能异常
| 子分类 | 关键词/场景 |
|--------|------------|
| T1-1 崩溃/闪退 | 闪退、崩溃、卡死、无响应、打不开 |
| T1-2 功能失效 | 按钮点不了、导出失败、保存失败、生成失败 |
| T1-3 登录异常 | 登录不上、账号异常、扫码失败、验证码收不到 |

#### T2 生成质量问题
| 子分类 | 关键词/场景 |
|--------|------------|
| T2-1 内容空洞 | 内容太水、废话、没用、浪费时间、全部重写 |
| T2-2 逻辑混乱 | 前后矛盾、逻辑不对、驴唇不对马嘴 |
| T2-3 数据错误 | 数字不对、统计数据造假、事实性错误 |
| T2-4 风格不匹配 | 不是我要的、风格不对、不符合场景 |

#### T3 模板相关
| 子分类 | 关键词/场景 |
|--------|------------|
| T3-1 模板缺失 | 没有xxx模板、找不到想要的模板、模板太少 |
| T3-2 模板质量差 | 模板丑、模板土、模板过时、配色难看 |
| T3-3 模板使用 | 怎么换模板、模板怎么编辑、模板如何调整 |

#### T4 性能问题
| 子分类 | 关键词/场景 |
|--------|------------|
| T4-1 加载卡顿 | 加载慢、等半天、一直转圈、卡住了 |
| T4-2 生成卡顿 | 生成太慢、等了10分钟还没好 |
| T4-3 导出卡顿 | 导出慢、导出一直转圈、导出失败 |

#### T5 账号与订阅
| 子分类 | 关键词/场景 |
|--------|------------|
| T5-1 付费问题 | 扣费异常、重复扣费、免费试用被收费 |
| T5-2 退款诉求 | 退款、退款申请、退钱、不值这个价 |
| T5-3 套餐咨询 | 价格太贵、多少钱、有没有优惠、学生价 |

#### T7 数据安全
| 子分类 | 关键词/场景 |
|--------|------------|
| T7-1 隐私担忧 | 我的PPT会不会被看到、隐私安全吗 |
| T7-2 导出格式 | 导出来格式乱了、PPT打不开、兼容性问题 |

## 二、智能路由规则

### 2.1 路由决策树

```
工单入口
    │
    ├──【情绪检测】负面情绪关键词 → 优先处理 + 升级标记
    │
    ├──【关键词匹配】
    │   ├── 含"崩溃/闪退/打不开" → [T1-1] → 最高优先级 → 技术支持组
    │   ├── 含"退款/退钱" → [T5-2] → 紧急 → 退款专项组
    │   ├── 含"账号/登录" → [T1-3] → 高优先级 → 技术支持组
    │   ├── 含"内容太水/废话/没用" → [T2-1] → 中优先级 → AI优化组
    │   ├── 含"逻辑不对/前后矛盾" → [T2-2] → 中优先级 → AI优化组
    │   ├── 含"模板丑/找不到模板" → [T3-1/2] → 中优先级 → 设计组
    │   ├── 含"卡顿/加载慢" → [T4-1/2/3] → 高优先级 → 性能优化组
    │   ├── 含"价格/太贵" → [T5-3] → 低优先级 → 销售转化组
    │   ├── 含"隐私/安全" → [T7-1] → 高优先级 → 安全合规组
    │   └── 含"导出格式乱" → [T7-2] → 中优先级 → 技术支持组
    │
    └──【未识别】→ 人工分类 + 补充关键词库
```

### 2.2 路由目标组

| 目标组 | 负责分类 | SLA响应时间 | 处理时长目标 |
|--------|---------|------------|-------------|
| **技术支持组** | T1, T4, T7-2, T8 | 15分钟 | 2小时 |
| **AI优化组** | T2 | 30分钟 | 4小时 |
| **设计组** | T3 | 1小时 | 8小时 |
| **账号客服组** | T5 | 10分钟 | 1小时 |
| **销售转化组** | T5-3, T6 | 2小时 | 24小时 |
| **安全合规组** | T7-1 | 30分钟 | 4小时 |

### 2.3 优先级升级规则

| 触发条件 | 升级动作 |
|---------|---------|
| 情绪关键词命中≥3个 | 优先级+1，自动抄送客服主管 |
| 用户提及"投资人/融资/BP" | 标记为VIP，优先处理 |
| 退款诉求 | 直接转退款专项组，2小时内必须响应 |
| 崩溃/闪退连续3次以上 | 标记为P0，触发技术紧急响应 |
| 同一用户7天内提交相同问题≥3次 | 触发工单合并+深度复盘 |

## 三、情绪路由联动

### 3.1 情绪敏感词权重

| 情绪等级 | 关键词示例 | 路由加成 |
|---------|-----------|---------|
| 🔴 高负面 | 垃圾、废物、骗子、坑人、浪费时间、愤怒、无语 | 优先级+2，触发情绪预警 |
| 🟠 中负面 | 失望、不好用、太差了、不满意、焦急 | 优先级+1，进入情绪跟进队列 |
| 🟡 低负面 | 希望改进、建议、能不能 | 正常处理，可选自动回访 |

### 3.2 情绪路由动作

```
情绪等级 = 🔴 高负面:
    → 工单优先级提升
    → 自动发送安抚消息模板
    → 抄送客服主管
    → 24小时内人工回访
    
情绪等级 = 🟠 中负面:
    → 工单保持正常优先级
    → 自动发送感谢反馈消息
    → 7天内自动问卷
    
情绪等级 = 🟡 低负面:
    → 正常流程处理
    → 标记为"建议型反馈"
```

## 四、工单处理流程

### 4.1 标准流程

```
用户提交 → 自动分类 → 情绪检测 → 路由分配 → 处理 → 用户评价 → 关闭
     │          │           │         │        │        │
     └──────────┴───────────┴─────────┴────────┴────────┴──→ 归档分析
```

### 4.2 闭环标准

| 分类 | 闭环标准 |
|-----|---------|
| T1 功能异常 | 用户确认功能恢复正常 |
| T2 生成质量 | 用户确认生成质量可接受 或 进入优化队列 |
| T3 模板相关 | 提供解决方案 或 确认加入模板需求 |
| T5 账号订阅 | 问题解决 或 退款完成 |
| T7 数据安全 | 解答用户隐私疑虑 或 确认导出成功 |
