# DemoPPT v2.4 情绪识别预警配置

> 本文档定义 DemoPPT 产品工单系统中的情绪识别规则和预警配置，用于监控用户反馈中的情绪信号并触发相应处理流程。

---

## 一、情绪等级定义

| 情绪等级 | 等级代码 | 情绪类型 | 典型情绪词 | 预警级别 |
|---------|---------|---------|----------|---------|
| Level 1 | `CRITICAL` | 愤怒/沮丧 | 愤怒、崩溃、气死、忍无可忍、太过分了 | 红色预警 |
| Level 2 | `HIGH` | 不满意/失望 | 失望、不满、差劲、失望透顶、浪费时间 | 橙色预警 |
| Level 3 | `MEDIUM` | 中性/一般 | 一般、凑合、还行、希望改进、无所谓 | 黄色预警 |
| Level 4 | `LOW` | 满意/感谢 | 感谢、满意、不错、挺好、赞 | 绿色（正常） |

---

## 二、情绪识别关键词库

### 2.1 CRITICAL 级别（红色）— 强烈负面词汇

| 关键词分类 | 关键词列表 |
|-----------|-----------|
| 侮辱性词汇 | 垃圾、废物、骗子、无良、流氓、黑心 |
| 极度负面情绪 | 崩溃、气死、忍无可忍、要疯了、砸电脑、卸载 |
| 强烈诉求 | 退款、全额退款、立刻退款、马上退款、不再使用 |
| 威胁性表达 | 投诉到底、315投诉、法院见、举报、曝光 |

**匹配策略：** 检测到任一关键词即触发 Critical 预警

### 2.2 HIGH 级别（橙色）— 中度负面词汇

| 关键词分类 | 关键词列表 |
|-----------|-----------|
| 不满意表达 | 失望、失望透顶、不满、差劲、太差了 |
| 时间浪费表达 | 浪费时间、毫无意义、完全没有用、根本不行 |
| 重复失败表达 | 又失败了、总是失败、一直有问题、无数次 |
| 负面对比 | 别人家都比你们好、竞品比你们强太多 |

**匹配策略：** 检测到任一关键词即触发 High 预警

### 2.3 MEDIUM 级别（黄色）— 中性/需关注词汇

| 关键词分类 | 关键词列表 |
|-----------|-----------|
| 一般性抱怨 | 一般、凑合、还行、希望改进、可以更好 |
| 疑问性表达 | 为什么这样？能不能改进？希望能... |
| 建议性表达 | 如果能...就更好了、建议...、希望增加... |
| 无奈表达 | 算了、随便吧、也只能这样了 |

**匹配策略：** 检测到3个及以上关键词时触发 Medium 预警（单个不触发）

### 2.4 LOW 级别（绿色）— 正面词汇

| 关键词分类 | 关键词列表 |
|-----------|-----------|
| 感谢表达 | 感谢、谢谢、辛苦了、多谢 |
| 满意表达 | 满意、不错、挺好、很好用、赞 |
| 鼓励表达 | 继续加油、希望越做越好、支持你们 |

**匹配策略：** 检测到任一关键词则该工单不进入预警队列

## 三、触发条件配置

### 3.1 重复提交触发（Repeat Submission）

| 条件 | 配置值 | 说明 |
|-----|-------|------|
| 同一用户同类问题提交次数 | ≥ 2次 | 同一用户提交相同类型（Type相同）工单2次及以上 |
| 时间窗口 | 7天内 | 7天内的历史工单纳入统计 |
| 触发动作 | 优先级+1档 + 标记「重复提交」 | 仅触发一次，再次提交不再叠加 |

**示例：**
- 用户A在5月1日提交"T2-AI生成质量问题"，5月3日再次提交"T2-AI生成质量问题" → 触发重复提交规则

### 3.2 强烈负面词汇触发（Strong Negative Keywords）

| 条件 | 配置值 | 说明 |
|-----|-------|------|
| 检测范围 | 工单标题 + 工单内容 | 全文本扫描 |
| 匹配模式 | 精确匹配/模糊匹配 | 精确：完整匹配关键词；模糊：包含即可 |
| 触发动作 | 情绪级别设为CRITICAL + 红色预警 + 自动升级至客服主管 | 无时间窗口限制 |

### 3.3 放弃表达触发（Churn Intent）

| 关键词 | 类别 | 触发动作 |
|-------|------|---------|
| "不用了" | 放弃使用 | 情绪级别设为CRITICAL + 标记「流失风险」+ 触发用户挽留流程 |
| "卸载" | App删除意向 | 情绪级别设为CRITICAL + 标记「流失风险」+ 触发挽留问卷 |
| "退订" | 订阅取消意向 | 情绪级别设为HIGH + 标记「退订风险」+ 通知运营团队 |
| "注销" | 账号注销意向 | 情绪级别设为CRITICAL + 标记「账号流失」+ 通知账号团队 |
| "转投" | 竞品转向意向 | 情绪级别设为HIGH + 标记「竞品流失风险」+ 通知产品团队 |

## 四、预警动作配置

### 4.1 预警级别与响应

| 预警级别 | 颜色标识 | 通知方式 | 响应时效 | 处理团队 |
|---------|---------|---------|---------|---------|
| Critical | 🔴 红色 | 电话+短信+站内信+邮件 | 15分钟内 | 客服主管 + 所属专项组 |
| High | 🟠 橙色 | 站内信+邮件 | 1小时内 | 客服组长 + 专项组 |
| Medium | 🟡 黄色 | 站内信 | 4小时内 | 客服组 |
| Low | 🟢 绿色 | 无需预警 | N/A | N/A |

### 4.2 预警通知模板

#### Critical 预警通知
```
【DemoPPT 紧急预警】
类型：情绪识别-Critical
用户：{user_id}
工单号：{ticket_id}
情绪检测：{detected_keywords}
工单类型：{ticket_type}
严重程度：{severity}
时间：{timestamp}
处理人：待分配
请立即处理。
```

#### High 预警通知
```
【DemoPPT 高优预警】
类型：情绪识别-High
用户：{user_id}
工单号：{ticket_id}
情绪检测：{detected_keywords}
工单类型：{ticket_type}
时间：{timestamp}
请{target_team}尽快处理。
```

### 4.3 自动触发流程

| 触发条件 | 自动执行动作 |
|---------|------------|
| Critical + 账号类问题 | 1. 通知安全组<br>2. 冻结相关账号排查<br>3. 客服主管30分钟内电话回访 |
| Critical + 支付类问题 | 1. 通知支付组<br>2. 财务介入确认<br>3. 优先处理退款流程 |
| Critical + 放弃表达 | 1. 触发用户挽留流程<br>2. 发放补偿优惠券（需主管确认）<br>3. 24小时内安排专人跟进 |
| High + 重复提交≥3次 | 1. 自动创建用户问题汇总档案<br>2. 标记为「顽固问题用户」<br>3. 升级至产品经理评估 |
| Medium + 同类问题≥5次 | 1. 创建问题分析工单<br>2. 通知产品团队<br>3. 纳入本周痛点热榜 |

## 五、情绪评分模型

### 5.1 基础评分规则

```
情绪得分 = 基础分 - 扣分项 + 加分项
```

| 项目 | 分值 | 说明 |
|-----|-----|------|
| 基础分 | 50分 | 所有工单起始分 |
| CRITICAL关键词 | -40分 | 检测到红色关键词 |
| HIGH关键词 | -25分 | 检测到橙色关键词 |
| MEDIUM关键词 | -10分 | 检测到黄色关键词（累计） |
| LOW关键词 | +10分 | 检测到绿色关键词 |
| 重复提交（2次） | -10分 | 同一类型第2次提交 |
| 重复提交（3次及以上） | -20分 | 同一类型第3次及以上提交 |

### 5.2 情绪等级换算

| 得分区间 | 情绪等级 | 预警级别 |
|---------|---------|---------|
| 0-20分 | CRITICAL | 🔴 红色 |
| 21-35分 | HIGH | 🟠 橙色 |
| 36-45分 | MEDIUM | 🟡 黄色 |
| 46-60分 | LOW | 🟢 绿色 |

## 六、特殊场景处理

### 6.1 讽刺/反语检测
- 当检测到 "真是太好了"、"太棒了"、"绝了" 等词汇时
- 系统结合上下文判断是否为反语（正面词但上下文为负面）
- 如判定为反语，强制升级为 CRITICAL

### 6.2 组合负面词汇
- 同时检测到"退款"+"垃圾"/"骗子" → 直接触发 Critical 并标记「高危投诉」
- 同时检测到"卸载"+"再也不用了" → 触发 Critical 并激活紧急挽留流程

### 6.3 沉默用户检测
- 注册后超过30天未登录用户首次提交工单 → 情绪评分额外扣10分
- 付费用户提交工单 → 情绪评分额外扣5分（高预期导致）

## 七、配置参数表

| 参数名称 | 默认值 | 可调整范围 | 说明 |
|---------|-------|----------|------|
| `critical_keyword_match_mode` | exact | exact/fuzzy | 关键词匹配模式 |
| `repeat_submission_window_days` | 7 | 1-30 | 重复提交统计窗口（天） |
| `medium_keyword_threshold` | 3 | 1-5 | Medium级触发关键词数量阈值 |
| `alert_escalation_timeout_minutes` | 15 | 5-60 | 预警超时升级时间（分钟） |
| `churn_trigger_keywords` | 见3.3 | 自定义 | 流失意图关键词列表 |

## 八、情绪识别报告

系统每日生成情绪识别报告，包含：

- 情绪趋势环比分析
