# DemoPPT v2.4 智能工单路由规则

> 本文档定义 DemoPPT 产品的智能工单路由规则，包含分类、优先级、处理团队和 SLA 响应时间。

---

## 一、工单类型分类

| 类型编号 | 工单类型 | 类型代码 | 说明 |
|---------|---------|---------|------|
| T1 | 账号/登录问题 | `ACCOUNT` | 注册、登录、密码找回、账号封禁等 |
| T2 | AI生成质量问题 | `AI_QUALITY` | 内容不符合预期、生成失败、生成速度慢等 |
| T3 | 模板相关问题 | `TEMPLATE` | 找不到模板、模板显示异常、模板加载失败等 |
| T4 | 导出问题 | `EXPORT` | 格式错位、导出失败、文件损坏等 |
| T5 | 支付/定价问题 | `BILLING` | 扣费异常、订阅变更、退款申请、价格咨询等 |
| T6 | 功能建议/Bug反馈 | `FEEDBACK` | 新功能建议、现有功能Bug、未归类问题等 |

---

## 二、优先级分级体系

| 优先级 | 等级代码 | 定义 | 典型场景 |
|-------|---------|------|---------|
| P0 - 紧急 | `CRITICAL` | 业务中断，用户完全无法使用 | 登录后直接崩溃、支付后未到账导致服务中断、大规模用户受影响 |
| P1 - 高 | `HIGH` | 功能严重受损，影响核心流程 | AI生成100%失败、导出文件损坏无法打开、账号被盗 |
| P2 - 中 | `MEDIUM` | 功能部分受损，存在 workaround | 生成结果偶尔不符合预期、模板加载慢、部分用户偶发问题 |
| P3 - 低 | `LOW` | 体验问题或轻微Bug | 界面显示轻微错位、功能建议、非关键性体验问题 |

## 三、路由规则矩阵

### T1 - 账号/登录问题

| 优先级 | 场景描述 | 处理团队 | 一线响应SLA | 解决SLA |
|-------|---------|---------|------------|---------|
| P0 CRITICAL | 账号被盗、账户异常、登录导致App崩溃 | 安全组 + 客服主管 | 15分钟 | 2小时 |
| P1 HIGH | 密码无法找回、账号被封禁申诉、登录异常 | 账号专项组 | 30分钟 | 4小时 |
| P2 MEDIUM | 登录提示模糊、第三方登录异常、账号设置问题 | 客服-账号组 | 2小时 | 8小时 |
| P3 LOW | 账号信息更新、多设备登录咨询、登录历史查询 | 客服-基础组 | 4小时 | 24小时 |

| P2 MEDIUM | 生成结果偶尔不符合预期、生成速度过慢（>30秒） | AI优化组 | 2小时 | 8小时 |
| P3 LOW | 生成风格微调请求、内容细节优化建议 | 产品组 | 4小时 | 24小时 |

## 五、路由决策流程

```
用户提交工单
     │
     ▼
┌─────────────────────────────────────┐
│ 1. 自动分类（NLP/关键词匹配）        │
│    → 识别工单类型（T1-T6）            │
└─────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────┐
│ 2. 情绪识别评分                      │
│    → 评估情绪等级（见情绪识别配置）    │
│    → 如为Critical，优先级+1档        │
└─────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────┐
│ 3. 历史工单检测                      │
│    → 同一用户同一类型工单≥2次        │
│    → 优先级+1档                      │
└─────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────┐
│ 4. 匹配路由规则                      │
│    → 确定处理团队                     │
│    → 确定SLA时间                      │
└─────────────────────────────────────┘
     │
     ▼
┌─────────────────────────────────────┐
│ 5. 升级机制                          │
│    → SLA未响应 → 自动升级至主管       │
│    → P0工单 → 同时通知部门负责人      │
└─────────────────────────────────────┘
```

## 六、特殊路由规则

### 6.1 情绪升级规则
- 情绪识别为 `Critical`（愤怒/沮丧）时，工单优先级自动提升1档
- 情绪识别为 `Critical` 且属于 P0 问题时，同时触发电话回访

### 6.2 批量问题升级规则
- 同一问题1小时内提交≥10次 → 自动升级为 P0，并创建应急预案
- 同一问题24小时内提交≥50次 → 通知产品/研发负责人

### 6.3 VIP用户规则
- 付费企业用户 → 优先级+1档
- 年度付费用户 → 优先分配至专项组处理

### 6.4 静默用户激活规则
- 用户超过30天未登录后首次提交工单 → 优先级+1档

## 七、SLA 计时规则

| SLA类型 | 计时起点 | 计时暂停条件 | 超时动作 |
|--------|---------|------------|---------|
| 一线响应 | 工单创建时间 | 无 | 自动升级+发送提醒 |
| 解决SLA | 工单创建时间 | 用户请求暂停等待（需确认） | 发送满意度调查+自动关闭 |
